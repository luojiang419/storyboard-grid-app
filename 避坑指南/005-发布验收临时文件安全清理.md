# 发布验收临时文件安全清理

## 现象

远端 Release 验收需要重新下载安装包与 `.sha256` 文件。最初使用随机临时目录并在 `finally` 中递归删除，命令被本机安全策略拒绝，验收脚本未执行。

## 原因

递归删除目标由运行时动态生成，即使位于系统临时目录，安全策略也无法在执行前可靠确认最终范围。

## 处理方式

1. 使用项目 `tmp` 目录下固定、带版本号且不存在的两个临时文件。
2. 下载前拒绝覆盖已有文件。
3. 验证 Release、manifest、asset digest、下载哈希和校验文件一致。
4. 确认绝对路径仍位于项目 `tmp` 目录后，逐文件删除，不执行递归目录清理。

## 后续规避

- 发布验收只创建数量确定、名称确定的临时文件。
- 删除前同时校验绝对父目录和文件名前缀。
- 不对动态目录执行递归删除，不使用通配符或未解析环境变量作为删除目标。
- 删除后再次检查文件不存在，并确认 Git 工作区没有残留改动。
