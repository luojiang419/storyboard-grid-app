# 组件测试图片夹具避免 Windows 临时文件锁

## 问题现象

故事板组件测试把 PNG 写入临时目录并实际渲染后，即使卸载 Widget、清理 Flutter 图片缓存，Windows 偶尔仍会在测试清理阶段抛出 `PathAccessException`，提示临时目录正被其他进程使用。

## 根因

- Flutter 测试引擎的图片解码链路可能短暂持有源图片文件句柄。
- 测试断言完成不等于原生解码器已经释放文件句柄。
- 盲目增加固定延时既不稳定，也会拖慢测试。

## 解决方式

当测试只需要一张有效图片、不会修改图片内容时，直接使用仓库内固定图片作为只读夹具；临时目录只保存测试数据库等可及时关闭的文件。测试结束时仍需卸载 Widget，并清理 `imageCache` 与 live images。

## 后续约束

- 需要验证文件增删、替换或归档时，继续使用独立临时图片。
- 只验证界面渲染、资源扫描或定位时，优先复用仓库内固定图片夹具。
- 不用长时间重试或吞掉临时目录删除异常来掩盖真实的文件句柄泄漏。
