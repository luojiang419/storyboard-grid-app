# 启动界面与工程管理开发方案

## 一、目标

软件启动后先进入欢迎页或工程首页；只有成功创建或打开工程，才能进入设计分镜图、多宫格裁切、故事板拼图、导出和设置等正式功能。

启动状态流：

```text
启动检查
  → 旧版数据安全接管
  → 欢迎页或首页
  → 创建/打开工程
  → 初始化工程会话
  → 正式操作界面
```

默认工程根目录为 `<软件目录>/data/project`。目录不可写时引导用户选择自定义位置并保存选择。工程内部的受管理文件只持久化相对路径，工程整体移动后仍可通过根目录中的 `project.storyboard` 恢复。

## 二、页面与交互

### 欢迎页

- 提供创建工程、打开工程、最近 5 个工程和进入首页。
- “进入首页”只跳过本次；另提供“下次启动直接进入首页”持久开关。
- 最近工程路径失效时可重新定位 `project.storyboard`。
- 支持拖入索引以及 `Ctrl+N`、`Ctrl+O`、`Ctrl+I` 快捷键。

### 工程首页

- 顶部提供创建、打开、导入、搜索、排序和默认根目录设置。
- 工程使用 300–360px 自适应卡片展示名称、路径、修改时间、存储位置和健康状态。
- 卡片主区域打开工程，右下角固定删除、迁移、导出按钮。
- 删除分为“仅移出首页”和需要输入工程名确认的“永久删除文件夹”。

### 创建与正式操作

- 工程名限制 1–80 个字符，拒绝 Windows 非法字符和保留名；同名目录自动建议递增名称。
- 创建过程先写临时目录，全部成功后原子改名，创建成功自动打开。
- 正式界面标题显示当前工程名，并提供返回首页；退出工程前保存状态、关闭数据库并释放文件锁。

## 三、工程格式与数据安全

```text
data/project/工程名称/
├─ project.storyboard
├─ database/project.sqlite
├─ imports/
├─ cuts/
├─ storyboards/custom_folders/
├─ generated_images/
├─ exports/
├─ temp/
└─ logs/
```

索引为 UTF-8 JSON，包含格式标识、结构版本、UUID、名称、创建/修改时间、相对数据库路径和可选封面路径。索引不保存工程绝对路径，所有相对路径都经过规范化并拒绝 `..`、绝对路径和目录逃逸。

全局数据库保存主题、API 配置、更新设置、欢迎页偏好、默认工程根目录、最近工程目录和旧版迁移标记；工程数据库只保存业务数据和工程页面状态。API Key 等全局配置不得进入工程压缩包。

### 迁移

采用“复制到临时目录—校验—登记新位置—删除旧目录”。若最终删除失败，保留新旧两份并明确提示，绝不破坏已验证的新副本。

### 导出与导入

- 导出为 `工程名_yyyyMMdd_HHmm.storyboard.zip`，包含工程索引、数据库、素材和导出成果。
- 排除 `temp`、`logs`、锁文件、SQLite WAL/SHM；包清单保存逐文件 SHA-256。
- 导入先在默认根目录解压到临时目录，防止 Zip Slip，校验清单、摘要、索引和数据库后原子落位。
- 工程 ID 重复时导入为副本，生成新 ID 和不冲突名称，不覆盖已有工程。

### 旧版接管

首次发现旧版业务数据时创建“旧版工程”。只迁移业务表和工程状态白名单，复制受管理素材并重写内部路径；校验行数、文件和数据库完整性后登记。原数据库与素材保持不变作为回退。

## 四、模块顺序

1. 工程基础设施：索引、目录、路径解析、最近项目仓储和会话基础。
2. 数据接入：全局/工程数据库分离、现有模块接入、旧版迁移。
3. 欢迎页与首页：启动门禁、卡片、创建/打开和路径修复。
4. 工程操作：迁移、二段式删除、ZIP 导入导出和失败恢复。
5. 全局集成：设置、文件关联、测试、Windows Release、安装包和缓存清理。

每个模块开始前递增创建阶段备份，完成后递增生成进度快照；只读取最新快照继续开发。

## 五、验收标准

- 未创建或打开工程时无法进入正式功能。
- 完整移动工程目录后，仅选择新位置的索引即可恢复全部受管理数据。
- 创建、迁移、导入或导出失败不破坏原工程，也不留下不可识别半成品。
- 工程包不包含全局密钥、更新配置、日志和临时缓存。
- 删除、迁移、导出交互明确，永久删除具备目录边界保护。
- 旧版接管前后业务数据、图片和工作区状态一致，原数据不被改动。
- 静态分析、现有测试、新增工程管理测试、Windows Release 和安装包构建全部通过。
